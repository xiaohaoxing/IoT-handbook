# 智能家居管理平台 + Agent 学习计划

## 学习目标

这套计划面向“产品/方案侧优先”的学习路径。学习完成后，你应该能做到：

- 说清楚智能家居管理平台的核心分层
- 判断哪些能力应该由 IoT 平台负责，哪些能力适合交给 Agent/RPA
- 和朋友讨论 PoC 范围、平台边界、接入方式和风险
- 设计一个可落地的“智能家居平台 + 实在Agent”方案草图

## 总体学习原则

先学边界，再学能力；先学架构，再学工具；先学怎么判断问题，再学怎么操作界面。

不建议一开始就沉到某个工具的按钮级细节里。你当前最需要建立的是“上下文框架”，否则后面容易把实在Agent 当成设备平台，或者把 IoT 平台当成普通后台系统。

## 学习阶段

### 第一阶段：建立上下文框架

目标：理解智能家居平台是什么、为什么要分层、每层负责什么。

顺序：

1. 平台分层与职责边界
2. 设备、属性、指令、事件这些基础对象
3. 典型业务流：控制、联动、告警、报修、运营
4. Agent/RPA 在整体架构中的位置

产出：

- 一张“智能家居平台分层图”
- 一份“哪些能力属于底层、哪些属于上层”的边界表

### 第二阶段：理解产品与方案设计

目标：知道一个可卖、可讲、可落地的平台方案通常怎么组织。

顺序：

1. 角色与用户：住户、运维、物业、运营、客服
2. 核心场景：设备控制、异常处理、工单流转、报表分析
3. 数据流与业务流
4. PoC 范围定义

产出：

- 一份 3 个核心场景的清单
- 一份 PoC 范围建议

### 第三阶段：补系统集成能力

目标：知道平台之间怎样互通，以及 Agent/RPA 该怎样接入。

顺序：

1. API、Webhook、JSON、鉴权
2. 标准接口集成与非标准界面自动化
3. 什么时候用 API，什么时候用 RPA
4. 风险点：权限、误操作、数据一致性、可追溯性

产出：

- 一张“集成方式选择表”
- 一份“控制类操作的安全约束建议”

### 第四阶段：理解实在Agent 的合适用法

目标：把实在Agent 放在正确的位置，不神化，也不低估。

顺序：

1. 它擅长什么
2. 它不适合做什么
3. 它怎样和现有 IoT 平台结合
4. 怎样设计一个小规模 PoC

产出：

- 一份“适合/不适合”判断清单
- 一个 3 场景 PoC 方案

### 第五阶段：进入实际试用

目标：在理解边界之后，再去看工具和文档，避免学偏。

顺序：

1. 安装并登录实在Agent
2. 试最简单的查询/导出流程
3. 看执行计划与流程节点
4. 尝试把一个业务动作做成标准流程

产出：

- 一个简单试用记录
- 一份“这个工具在我们场景里到底值不值得用”的判断

## 建议节奏

如果按两周节奏推进，建议如下：

### 第 1 周：建立认知边界

- Day 1：学习平台分层与边界
- Day 2：学习设备模型与基础对象
- Day 3：学习核心业务流
- Day 4：学习 IoT 平台与运营平台差异
- Day 5：学习 Agent/RPA 的定位
- Day 6：整理分层图与边界表
- Day 7：复盘并和朋友做一次非技术讨论

### 第 2 周：进入方案与落地视角

- Day 8：学习角色与场景设计
- Day 9：学习 PoC 范围定义
- Day 10：学习 API / Webhook / JSON 基础
- Day 11：学习 API 与 RPA 的选型边界
- Day 12：学习实在Agent 在整体方案中的位置
- Day 13：设计 3 个 PoC 场景
- Day 14：输出一版方案草图

## 推荐学习顺序

你接下来按这个顺序学：

1. 第 1 册：平台分层与边界
2. 第 2 册：设备模型与核心对象
3. 第 3 册：核心业务流
4. 第 4 册：IoT 平台与运营平台的区别
5. 第 5 册：Agent/RPA 的定位与边界
6. 第 6 册：角色、场景与 PoC 设计
7. 第 7 册：API / Webhook / JSON 基础
8. 第 8 册：API、RPA、Agent 的选型方法
9. 第 9 册：实在Agent 与智能家居方案结合方式
10. 第 10 册：PoC 方案输出模板

## 当前学习入口

现在先学第 1 册，不要先跳到工具操作。

你学完后，应该能回答这 3 个问题：

1. 智能家居管理平台至少分几层？
2. 哪些能力必须放在底层 IoT 平台？
3. 为什么实在Agent 更适合做上层能力而不是底层控制核心？
